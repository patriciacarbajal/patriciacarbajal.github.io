Enumerable Methods - enumerable#group_by

September 16, 2014

Today I'm going to discuss a useful Ruby method - Enumerable#group_by.

This method takes in a sequence of enumerable values, puts them through a block, and produces a hash. The keys of the new hash are the evaluated results from the block. The values of the hash are arrays of the elements in the original sequence that correspond to the key. If no block is passed, the method will return an enumerator.

This must all sound very confusing. Allow me to demonstrate with an example...

array = [1, 2, 3, 4, 5, 6]

array.group_by { |x| x % 2}


The result will be:

#=> {1=>[1, 3, 5], 0=>[2, 4, 6]}



In the above example, array is the sequence of enumerable values (objects). Each object of the array can be passed through the method one by one.

In the block, x represents each object being passed through the method. Each one of them is evaluated through the expression x % 2. Each unique result of this evaluation now becomes a key in the resulting hash.

The value for each key is an array with the elements for which the result is true. In this example, 1 % 2, 3 % 2 and 5 %2 each result in 1, and 2 % 2, 4 % 2 and 6 % 2 each result in 0.



And there you have it!
