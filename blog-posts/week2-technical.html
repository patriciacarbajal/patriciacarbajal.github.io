Relative, Absolute, and Fixed Positioning - What's the difference?

September 6, 2014

Relative, absolute, and fixed positioning can be a source of confusion when learning CSS. Let's see if we can clear it up.

First of all, you should know that all elements have the position "static" by default. You won't need to use this unless you are trying to remove positioning for some reason. Let's not worry about it for now.

Relative positioning means that the element is positioned relative to itself. In order for this to have an effect, you must set another positioning attribute (such as top, right, bottom and left). For example,

position: relative;
left: 20px;

would shift the element 20px to the right from where it would normally be on the page.

When the element is shifted, it's original space is preserved in the flow of the page so other elements won't fill it's spot.

Absolute positioning, on the other hand, lets you position an element anywhere on the page by using the top, right, bottom, and left positioning attributes.

Position: absolute;
top:0px;
right:0px;

will place the element at the top right corner of the page.

However, if an element is within a parent element that is set to position: relative;, absolute positioning will set the child element relative to the parent element.

Remember that unlike relative positioning, the element's space will not be preserved in the flow of the page when shifted. Other elements will fall into it's original place.

Fixed positioning sets an element's location relative to the browser window, and keeps it there. When the page is scrolled, the element will remain fixed in the same position as the rest of the page shifts as usual.

Hopefully you now better understand the difference between absolute, relative and fixed positioning!
